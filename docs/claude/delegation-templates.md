# 派工 Prompt 模板

寫於 2026-07-10。配合 dispatch.md 使用：三件套（目標與動機／驗收條件／回報格式）
缺一不派。`{...}` 是填空。每個模板都標了建議的 subagent_type 與 model。

通用規則：
- prompt 內講清楚專案根目錄與環境（agent 是冷啟動，什麼都不知道）。
  最低限度附上：「專案在 c:\Users\Daniel Lin\Desktop\ic_products_dashboard，
  Python 用 ./ic_venv/Scripts/python.exe，先讀 CLAUDE.md」。
- 回報一律要求「結論先行 + 檔案:行號」，長產物落檔傳路徑。

---

## 1. 搜尋 / 定位（Explore, model 省略即可）

```
在專案 {根目錄} 中找：{要找什麼，例如「gross_margin_pct 的所有計算位置」}。
背景：{為什麼要找，例如「要改公式，需要確認影響面」}。
搜尋廣度：{quick|medium|very thorough}。
驗收條件：列出所有匹配位置，不能只給第一個。
回報格式：每行一筆「檔案:行號 — 一句話說明該處在做什麼」，最後一行給總數。
找不到也要明說「查無」與已排除的目錄，不准編造。
```

## 2. 實作（general-purpose, model: sonnet）

```
專案：c:\Users\Daniel Lin\Desktop\ic_products_dashboard。先讀 CLAUDE.md 的
「環境實況」與「目錄結構與三個邊界」，遵守分層（src/ 不碰 Streamlit）。

目標：{做什麼}
動機：{為什麼做，主管/使用者要解決什麼}
範圍內：{明確要改的檔案或模組}
範圍外：{明確不准動的，例如「不改 schema.py、不動其他頁面」}

驗收條件（全部滿足才算完成）：
- ./ic_venv/Scripts/python.exe -m pytest tests/ -q 全綠
- {功能特定條件，例如「新函式對空 DataFrame 回傳空表不 raise」}
- 新公式已在 tests/ 補測試

回報格式：改了哪些檔（檔案:行號區間）、測試輸出末三行原文、
任何偏離指示的地方與理由。不要貼整個 diff。
```

## 3. 重構（general-purpose, model: sonnet；跨 >3 檔的不變量 → opus）

```
專案：{根目錄}，先讀 CLAUDE.md。

目標：把 {現狀} 重構為 {目標形狀}。
動機：{為什麼值得重構}
不變量（重構前後必須完全一致的行為）：
- {例如「所有 pytest 通過且不修改任何測試檔」}
- {例如「頁面 import 的公開函式簽名不變」}

步驟要求：先跑一次測試記錄基線，改完再跑一次，兩次結果都要回報。
驗收條件：測試基線不退步 + {結構性條件，例如「src/ 內 grep 不到 import streamlit」}。
回報格式：移動/改名清單（舊路徑 → 新路徑）、測試前後輸出末行、
沒動但發現有問題的地方列成「遺留問題」清單。
```

## 4. 研究 / 查證（general-purpose, model: sonnet）

```
問題：{要回答什麼}
動機：{這個答案會決定什麼}
資訊來源優先序：{例如「先查官方文件，再查 repo 內用法，最後才是通用知識」}
驗收條件：每個結論都要附來源（URL 或 檔案:行號）；查不到的明確標註「未查證」，
禁止用訓練記憶充當查證結果。
回報格式：先一段直接回答，然後條列證據。若答案是「不確定」，
列出已排除的假設與下一步建議。
```

## 5. 審查（general-purpose, model: sonnet；獨立性要求見下）

```
審查對象：{檔案清單或 diff 範圍}
背景：{這批改動要達成什麼}——但請獨立判斷，不要因為知道意圖就放水。
審查重點（按序）：
1. 正確性：會不會在 {具體情境，例如「空篩選、單月資料」} 下出錯
2. 邊界違規：src/ 是否 import streamlit、金額是否非 USD 儲存
3. 規則互相打架、路徑或指令與現實不符（對照 CLAUDE.md 環境實況）
4. 弱模型會誤讀的模糊語句（若審查對象是文件）

驗收條件：每個 finding 附 檔案:行號 + 一句失敗情境（什麼輸入會觸發什麼錯誤）。
沒有 finding 就明說「未發現」，不要硬湊。
回報格式：按嚴重度排序的清單；不確定的標 [PLAUSIBLE]，確認過的標 [CONFIRMED]。
```

**獨立性要求**：審查 agent 必須是 fresh context——不能是寫那份產出的同一個 agent
（用 SendMessage 追問寫作者「你檢查一下」不算審查，見 dispatch.md「驗證不自驗」）。
