# 判斷力 Rubrics — 弱模型可執行版

寫於 2026-07-10。每條判準附正例／反例，多數取自本專案真實 session。
遇到 rubric 沒涵蓋的情況：先看 §6 誠實條款，不要硬套。

## 1. 何時該升級模型

**判準**：滿足任一 → 升級（路徑見 dispatch.md）：
- 同一子任務用了兩種**不同方法**都失敗
- 任務需要同時保持 3 個以上檔案的不變量（跨檔重構、併發、schema 遷移）
- 錯誤訊息看得懂每個字但組合起來無法解釋

**正例**：修一個 race condition，第一次加 lock 沒用、第二次改事件順序也沒用，
兩次的理論互相矛盾 → 帶兩份失敗軌跡升 opus。
**反例**：測試掛了，原因是 fixture 少一個欄位（本專案 test_risk.py 曾缺 `cogs_usd`）——
這是看得出來的機械修補，補上就好，升級是浪費。

## 2. 何時算真的完成

**判準**：全部滿足才算：
- [ ] 驗收條件逐條核對過（不是「大概都做了」）
- [ ] **實跑過**：測試輸出貼得出來、或程式實際執行過看到結果
- [ ] 產出數字經過量級合理性檢查（毛利率不會是 300%、成本不會是負的）
- [ ] 落地協議做完：TODO.md 勾選更新、CLAUDE.md 若受影響已同步（diagnosis.md §1）

**正例**：本專案加 risk analytics 時：pytest 18 全過 + 重生資料後實算一輪，
確認 28nm 成本 > 130nm、HHI 數字落在合理區間，才回報完成。
**反例**：「檔案寫好了、py_compile 過了」就回報完成——compile 過只代表語法對，
`quantity` 欄位若不存在要到 runtime 才炸。

## 3. 何時該停下來問使用者

**判準**：滿足任一 → 停下來問，其他情況自己決定並在回報中說明選擇：
- 動作不可逆或對外（刪檔、push、寄出、對外發布），且沒有明確授權
- 涉及密碼、金鑰、憑證的產生或處理方式
- 發現使用者的描述與 repo 現實矛盾（說有的檔案不存在、說沒改的檔案被改過）
- 兩條路會導致**不同架構**，之後難回頭（SQL vs CSV、換 auth 方案）

**正例**：建 admin 帳號需要密碼——密碼是使用者的祕密，必須問，不能自己編一個。
**反例**：表格預設排序用銷量還是營收——選一個合理的、在 UI 加切換、回報時提一句。
問這種問題是把決策成本轉嫁給使用者。

## 4. 什麼訊號代表方向錯了（該換路，不是重試）

**判準**：出現任一 → 停止重試，換方法或問人：
- 同一指令原樣重跑第二次，還在期待不同結果
- 每修一個錯就冒出兩個新錯
- 開始對抗框架：需要越來越多 hack 才能讓方案活著
- 修正的理論解釋不了上一次為什麼失敗

**正例**：`echo pw | python init_admin.py` 卡死——Windows getpass 讀 console 不讀 stdin，
換 pipe 寫法一百次都一樣。正確反應：殺掉、改寫非互動腳本（換路），而不是試
`printf` / heredoc / PowerShell 版 pipe（同路重試）。
**反例**：網路請求 timeout 一次就整個換方案——單次 timeout 可能是暫時性的，
重試一次是合理的；連續兩次才構成換路訊號。
**適用範圍釐清**：「重試一次」只適用於**暫時性外部失敗**（網路、檔案鎖）。
若失敗的是 subagent 的能力（haiku 答錯），走 dispatch.md 升降級——直接升級，
不重試同一顆模型。兩條規則管的是不同東西，不要混用。

## 5. 品質底線怎麼驗

交付前的最小 checklist（本專案版）：
- [ ] `./ic_venv/Scripts/python.exe -m pytest tests/ -q` 全綠
- [ ] 新 analytics 函式：用 `data/raw/` 真資料實算一輪，人眼掃數字量級
- [ ] UI 改動：空篩選結果（`fact.empty`）路徑不炸
- [ ] 沒有祕密（密碼、COOKIE_KEY）出現在 code、log、或回報文字裡
- [ ] `src/` 沒有 import streamlit（邊界一，`grep -r "import streamlit" src/` 應為空）

## 6. 誠實條款——rubric 補不了的東西

拆解、驗證、多樣本評審能補**執行品質**；補不了**模糊題與品味**：
- 命名品味、UX 布局取捨、「這個分析主管會不會覺得有用」——沒有機械判準。
- 遇到時的正確動作（按序嘗試）：
  1. 給使用者 2 個具體選項 + 各自 trade-off（一句話），讓他選；
  2. 值得的話升級模型拿第二意見；
  3. 都不行就明說「這題是品味判斷，我選了 X，理由 Y，不確定是否符合你的偏好」。
- **禁止**：假裝模糊題有客觀答案、或編造「業界慣例」背書自己的猜測。
  查不到的就標註查不到。
